Les erreurs fréquentes

Confusion entre opérateur d'affectation et d'égalité.

if (val=29)
//devrait être :
if (val == 29)

Problème de priorité des opérateurs.

while(carac = _getch() != '\r') devrait se lire
// L'opérateur a priorité sur celui d'affectation, la valeur de carac sera 0 ou 1 et non la valeur lue.
// La bonne façon :
while((carac = _getch()) != '\r')

Erreurs d'indices dans les variables dimensionnées.


void main(void)
{
int tableau[4];
int Compte:;
for(Compte=1; Compte > 4; Compte++) //L'indice du tableau doit débuter à 0.
...
}

Omission d'un indice dans un tableau multidimensionnel

tableau[0,1]
//devrait se lire
tableau [0][1]

Dépassement des limites d'un tableau

Tout indice invalide (<0 ou >= au nombre d'éléments) provoque une erreur qui peut être difficile à détecter.

Confusion entre constantes caractères et chaînes de caractères

Une constante est placée entre '', tandis qu'une chaîne de caractères est placée entre des guillemets.

Valeur retournée par une fonction de bibliothèque

Il est important de vérifier la valeur de retour d'une fonction de la bibliothèque.

Double emploi de noms de fonctions de bibliothèque

Afin d'éviter la confusion, il est prudent de ne pas nommer une fonction personnelle avec le même nom qu'une fonction de la bibliothèque.

Omission des fichiers d'en-tête pour les fonctions de bibliothèque

L'utilisation d'une fonctionnalité d'une fonction de la bibliothèque nécessite de l'inclure pour ne pas avoir d'erreur.

using namespace std;
void main(void) {
cout << "Bonjours Monde"
}
// Pour un code fonctionnel : #include<iostream>
using namespace std;
void main(void) {
cout << "Bonjours Monde"
}

Else non associé au bon if

Sans accolade, le est est toujours associé au dernier if qui le précède.

// on veut un programme qui dit de ne pas aller à l'école si malade, mais ajoute également une information additionnelle si nous avons de la fièvre
if (EstMalade == true){
if(EstFievreuse == true) {
cout << "Boire une tisane de gingembre";
}
else
cout << "Ne pas aller à l'école";
}
// dans ce cas, il serait préférable d'écrire
if (EstMalade == true){
if(EstFievreuse == true) {
cout << "Boire une tisane de gingembre";
}
}
else
cout << "Ne pas aller à l'école";

Omission des doubles barres inversée dans les spécifications d'un chemin d'accès

c:\temp\numero.txt devrait être c:\\temp\numero.txt

Omission de l'instruction break après un case de l'instruction switch

Omission des << dans un cout

Accolades {} mal placées ou manquantes

Oubli des parenthèses lors de l'appel à une fonction.

clrscr; devrait être clrscr();
