Les types créés par l'utilisatrice

Selon Stroustrup, la clé du C++ réside en partie dans la possibilité de créer des variables par les programmeur.e.s.

A struct (a structure) is a sequence of elements (called members) of arbitrary types.
• A union is a struct that holds the value of just one of its elements at any one time.
• An enum (an enumeration) is a type with a set of named constants (called enumerators).
• enum class (a scoped enumeration) is an enum where the enumerators are within the scope
of the enumeration and no implicit conversions to other types are provided. Bjarne Stroustrup, in The C++ Programming Language, p.202

Les structures

Une structure est un type informatique créé par l'utilisatrice (user-created)
Pour définir une structure :

Définitions du type d'une structure et la définition des variables de cette structure.

  1. La définition du type de la structure.
  2. La définition des variables de la structure. Ce n'est que lors de la définition des variables que l'espace mémoire est réservé.

Cas 1 : Définitions simultanées (structure et variable)

struct NomDeLaStructure_s //NomDeLaStructure_s est de type structure
{
    type NomDuMembre_1;
    type NomDuMembre_1;
    type NomDuMembre_2;
    …
    type NomDuMembre_N;
} VarStruct; //VarStruct est une variable de type struct NomDeLaStructure_s.

Cas 2 : Définitions indépendantes

struct NomDeLaStructure_s //définition de la structure. Par convention nomme une structure en finissant avec le suffixe '_s'
{     type NomDuMembre_1;
    type NomDuMembre_1;
    type NomDuMembre_2;
    …
    type NomDuMembre_N;
}; struct NomDeLaStructure_s VarStruct; //VarStruct est une variable de type NomDeLaStructure_s

Cas 3: Définitions simultanées, sans nom de type

struct
{
    type NomDuMembre_1;
    type NomDuMembre_1;
    type NomDuMembre_2;
    …
    type NomDuMembre_N;
} VarStruct;

Dans ce cas, il faut absolument déclarer chaque des variables de ce type lors de la définition

Cas 4 L Définition d'un type structure avec typedef, pour du C pur

struct NomDeLaStructure
{
    type NomDuMembre_1;
    type NomDuMembre_1;
    type NomDuMembre_2;
    …
    type NomDuMembre_N;
} typedef struct NomDeLaStructure NomDeLaStructure_s;
NomDeLaStructure_s NomDeLaStructure;

Conventions de codage

Règle 1 :
Les déclaration des types structures devront être faites avant la fonction main()
Règle 2 :
Lorsqu'il y a trop de déclaration de type, mieux vaut les consignées dans un fichier d'en-tête.
Règle 3 :
L'identifiant pour les types-structures se terminera par le suffixe_s
Règle 4 :
Les variables de type structure devront être déclarées à l'intérieur de la fonction main() ou de toutes autres fonctions du programmeur.
Règle 5 :
La déclaration d'une variable de type structure déclarée au niveau global (à l'extérieur du main()) devra être justifiées

L'affectation d'une variable structure

MonCompteBanque.NumeroCompte = 2345; //l'opérateur '.' signifie "élément de..."
MonCompteBanque.NomCompte = "JeanPol";
MonCompteBanque.Adresse.NoCivique = 255 ;
MonCompteBanque.Adresse.Rue = "Ontario est"
MonCompteBanque.Adresse.CodePostal = "H2X 1X6"
MonCompteBanque.Telephone = "(514)-982-3437";
MonCompteBanque.NumeroCompte = 201.0;

Assignation des valeurs d'une structure à une autre structure

struct MembreEquipe_s
{
    string NomJoueur;
    int NumeroJoueur;
} ; MembreEquipe_s CapitaineJaune, CapitaineVert;

CompteBanque_s MaBanque[50] //MaBanque est un tableau de 50 structure CompteBanque_s
