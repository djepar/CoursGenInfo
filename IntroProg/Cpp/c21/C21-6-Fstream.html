Les fichiers bufferisés et les fichiers non-bufferisés

Les fichiers bufferisés
Plus grande gamme de commandes et sont plus faciles à utiliser. L'utilisation de ces fichiers élimine la nécessité de définir les mémoires tampons dans lesquelles le contenu des fichiers est stocké.
Les fichiers non-bufferisés
Il faut définir la mémoires-tampons et s'occuper de l'intégrité des données lors que nous utilisont ce type de fichier.

dans le cadre de ce cours nous aborderons seulement des fichiers bufferisés.

Le mode de texte et le mode binaire

Mode texte
L'ensemble des données numériques sont converties en caractères ASCII
Mode binaire
L'ensemble des données sont converties en caractères binaire

La présentation des données dans les fichiers bufferisés

Les données peuvent êtres lues et écrites dans un fichier bufferisé de la même manièere que pour le flux cin/cout

Les types de io::

Ios::in
Ouvrir le fichier, il doit exister (if not fail())
ios::out
Ouvrir pour écrire. Si le fichier n'existe pas, il sera créé. Si le fichier existe déjà, il sera réécrit.
