2.1 Les opérations de sortie

On trouve plusieurs types d'opérations de sortie pour la programmation : des affichages à l'écran, des rapports imprimés, des chèques de paie ou encore des factures(p. 33)

2.1.1 La commande de sortie dans les diagrammes d'action

La commande de sortie est : ÉCRIRE.

2.1.2 Le contrôle de la mise en page

Une séquence d'échappement permet de produire une mise en page en C et C++

Séquences d'échappement:
\n passer à la ligne suivante;
\t aller à la prochaine tabulation;
\" un guillemet;
\\ un barre oblique inversée;
\r retour au début de la ligne(touche<enter>)
\b retour arrière(une colonne vers la gauche)
\a faire un bip!
\xhhh le caractère ASCII représenté par le nombre hhh(hexadécimal)
\ooo Le caractère ASCII représenté par le nombre ooo(octar
\0 le caractère nul

Il est également possible d'utiliser endl à la place de \n dans le flux cout, toutefois il y a une différence (LAQUELLE??????????)

2.1.3 Le flux COUT

Le flux cout affiche à l'écran n'importe quel type du C++ autant les string, char que les valeurs numériques.

Pour afficher le flux COUT ou faire tout autre opération nécessite le stream input-output, il faut inclure #include<iostream>

2.1.4 Les manipulateurs

Il est possible d'utiliser des manipulateurs avec la directive (#include<iomanip>)

Les manipulateurs:
setw(n): pour spécifier une largeur minimale de n caractères. On complète avec des espaces
oct: pour écrire des entiers en octal
hex pour écrire des entiers en décimal
dec: pour écrire en décial (par défault)
setprecision(n) pour préciser n décimales pour les réels et n chiffre à afficher pour les entiers.
fixed pour activer la notation en virgule flottante des réels.
scientific: pour activiter la notation scientifique des réels
left : pour activer le cadrage à gauche
left pour activer le cadrage à gauche
right pour activer le cadrage à droite (par défault)
endl change de ligne comme pour \n, toutefois il y a une difference (LAQUELLE)
À l’exception de setw(), les manipulateurs modifient les caractéristiques de toutes les prochaines valeurs affichées. Les réglages durent donc jusqu’à un nouveau réglage fait par ces mêmes manipulateurs. (p. 36)

STD

2.2 Les opérations d'entrée

2.2.1 La commande d'entrée dans les diagrammes d'action

Par convention, une opération d'entrée s'écrit LIRE dans les diagrammes d'action.

2.2.2 Le flux d'entrée cin

La lecture à partir du flux cin est dite «bufferisée» car elle se sert d’un tampon où les données sont stockées avant d’être affectées aux variables. Les données ne seront affectées aux variables qu'après avoir pressé la touche .

Le flux cin permet de lire des données de tous les types prédéfinis (int, float, char, string, etc.). C’est l’opérateur >> (opérateur lire) qui sera utilisé pour transmettre les données vers les variables(p. 45) Pour faire une entrée de donnée avec CIN, il faut :
  • supprimer les espaces initiaux,
  • mettre un espace comme séparateur entre les valeurs à lire. (on entend par espace : la touche espace, la tabulation (\t) et enter (\n)
  • Presser la touche <ENTER>
e la lecture à partir du flux cin se poursuit tant que les données entrées sont correctes. On s'arrête dès qu'un caractère invalide est rencontré. (p. 88)

2.2.3 Les fonctions _getche() et _getch()

From https://docs.microsoft.com/en-us/cpp/c-runtime-library/reference/getch-getwch?view=msvc-170, visité le 6 janvier 2021
Syntax en C :
int _getch( void ); wint_t _getwch( void ); Remarks : The _getch and _getwch functions read a single character from the console without echoing the character. None of these functions can be used to read CTRL+C. When reading a function key or an arrow key, each function must be called twice; the first call returns 0 or 0xE0, and the second call returns the actual key code.
These functions lock the calling thread and are therefore thread-safe. For non-locking versions, see _getch_nolock, _getwch_nolock.
By default, this function's global state is scoped to the application. To change this, see Global state in the CRT.

2.2.4 La fonction getline()

Getline permet de lire une chaîne de caractères qui comprend des espaces.

La fonction getline() lira une chaîne vide dans le cas où il reste un \n dans le tampon de lecture. (L'Italien, p.55)

2.3 Les fonctions gotoxy() et clrscr()

Deux fonctions de la bibliothèque maison cvm.h servant à faciliter la mise en page de programme

La gotoxy() permet de déplacer le curseur à l'écran n'importe où sur l'écran.
La syntaxe de gotoxy :
gotoxy(colonne, ligne)
ligne = n;
col = n;

La fonction clrscr() efface tout sur l'écran de sortie.

La fonction MessageBoxA()

MessageBoxApermet de créer facilement une boîte de message comprenant : titre, message, bouton.s et icône.

Le fichier en-tete requis pour cette fonction est #include<windows.h>
La syntaxe de cette fonction est : int MessageBoxA(Fenêtre Parent, Texte, Titre, Style);

Description des paramètres de MessageBoxA
Fenêtre Parent : Identifie la fenêtre parent de la fenêtre message. Si ce paramètre est NULL, c'est qu'il n'y a pas de parent.
Texte : Le message affiché.
Titre : Le titre affiché.
Style : Spécifie le contenu de la fenêtre de message.On peut y ajouter : nombre de bouton, bouton actif, icône et comportement

Les options de style de MessageBoxA
Nombre de bouton :
MB_ABORTRETRYIGNORE : Affiche trois boutons: Abandonner, Réessayer et Ignorer.
MB_OK : Affiche un bouton : OK
MB_OKCANCEL : Affiche deux boutons: OK et annuler
MB_RETRYCANCEL : Affiche deux boutons: Réessayer et annuler.
MB_YESNO : Affiche deux boutons: Oui et non.
YESNOCANCEL : Affiche trois boutons: Oui, non et Annuler.
Bouton actif
MB_DEFBUTTON1 défini le premier bouton comme celui qui est actif.
MB_DEFBUTTON2 défini le deuxième bouton comme celui qui est actif.
MB_DEFBUTTON3 défini le troisième bouton comme celui qui est actif.
Icône :
MB_ICONEXCLAMATION l’icône «!» (point d’exclamation)
MB_ICONINFORMATION l’icône «i» minuscule
MB_ICONQUESTION l’icône «?» (point d’interrogation)
MB_ICONSTOP l’icône «X» (stop)
comportement :
MB_SYSTEMMODAL la fenêtre restera visible jusqu'à réponse de l'usager
Description de la valeur de retour :
IDABORT le bouton Abandonner a été sélectionné
IDCANCEL le bouton Annuler a été sélectionné
IDIGNORE le bouton Ignorer a été sélectionné
IDNO aucun bouton n’a été sélectionné
IDOK le bouton OK a été sélectionné
IDRETRY le bouton Réessayer a été sélectionné
IDYES le bouton Oui a été sélectionné
