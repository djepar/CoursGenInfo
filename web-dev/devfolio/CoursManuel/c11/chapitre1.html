1.1 La programmation

Le cours s'articule en deux facettes :
"L'élaboration d'algorithmes simples"
"Le codage de ces algorithmes en langage C++"

L'informatique

  • Pour L'Italien, l'informatique consiste à résoudre un problème en trouvant une solution qui utilise l'ordinateur
  • Pour CNRTL, l'informatique est un [e]nsemble des opérations permettant la conception, la réalisation, le test, la maintenance de programmes d'apr. Morvan Informat. 1981 Programmation en langage machine, en langage assembleur, en langage évolué. La programmation inclut les travaux d'écriture et de mise au point des programmes (Le Garff1975)


Il n'y a pas de méthode universelle en programmation, mais nous pouvons toutefois faire un portrait général inspirés des lignes directrices communes aux différentes méthodes.

  1. Analyser le problème
    • Comprendre la problématique et les objectifs
    • Regrouper les informations et définir les grandes lignes de la solution. Notez bien que ce processus doit ce faire dialectiquement en collaboration avec l'usager.ère. 
    • Retravailler la solution jusqu'a ce qu'elle soit acceptées
  2. Élaborer la solution
    • Détailler et améliorer la solution.Le résultat obtenue de cette étape est un algorithme
  3. Programmer et documenter la solution
    • Choisir le langage informatique approprié
    • Traduire l'algorithme dans le langage choisi tout en documentant constament de programme

  1. L'en-tête : doit contenir le nom de l'auteur.trice, la date, le nom du programme et une description.
  2. Les directives au préprocesseurs : pour inclure les fonctions dans un programme C++, il faut inclure un '#include' puis mettre le fichier entre < > pour chaque fichier du préprocesseur.
  3. Les types définis par lxs programmeur.euse.s: cette section du code permet de définir de nouveau type qui ne sont pas définit par défaut.
  4. La déclaration des fonctions: les fonctions sont déclarées ici (prorotype de la fonction). Le prototype inclut : le type de sa valeur de retour, son nom et le type et le nom de ses paramèetres. La description détaillé des fonctions se fera dans la section 6.

1.5 Les données

1.5.1 L'espace-mémoire

Chaque variables, constantes ou données nécessitent un espace réservé par l'ordinateur, ce qu'on appelle un espace-mémoire

Pour bien comprendre l'espace mémoire, il faut être capable de :

  • "diff/rencvier les espaces-mémoires les uns des autres;
  • localiser un espace-mémoire;
  • mettre une donnée dans un espace-mémoire;
  • retrouver la valeur d'une donnée d'un espace-mémoire particulier;
  • modifier la valeur d'une donnée existante d'un espace-mémoire;" (p. 11)

Voici les éléments nécessaires d'un espace-mémoire proposés par les concepteur.trice.s de compilateurs :

  • nom unique
  • type afin de déterminer la taille
  • valeur défini (qui peut être modifié si il s'agit d'une variable

1.5.2 Règles d'écritures des noms de variable et constante

  • Doit être composé uniquement de caractères alpha-numériques et d'underscore. (Dans visual code, il est également possible d'utiliser les accents.
  • Ne pas commencer par un chiffre.
  • Sensibles aux majuscules et minuscules
  • Par convention : les constantes sont en majuscules tandis que les variables commencent avec une majuscule à chaque mot.
  • Ne pas commencer un nom avec un underscore
  • Ne pas utiliser des mots-réservés du langage et du compilateur.

1.5.3 La déclaration des variables

En C la déclaration de variables se fait toujours en début de fonction, tandis qu'en C++ les déclarations se font n'importe où dans le corps de la fonction

void main (void)
{ //Debut de main()
int Somme; //Déclarations des variables
short i, Compteur; // réserve un espace-mémoire nommé 'i' et 'Compteur' de 2 octets chacun
float Moyenne, Note; // réserve un espace-mémoire nommé 'Moyenne' et 'Note' de 4 octets chacun
double Distance, Salaire; // réserve un espace-mémoire nommé 'Distance' et 'Note' de 8 octets chacun
char Lettre; // réserve un espace-mémoire nommé 'Lettre' de 1 octet.

... //Corps de la fonction main()
... //Fin de la main()

}

1.6 Les opérateurs

Les opérateurs sont des mots ou des symboles qui indiquent des opérations spécifiques que les programmes devront exécuter
  • les opérateurs d'arithmétiques;
  • les opérateurs d'affectation;
  • les opérateurs d'incrémentation et de décrémentation;
  • les opérateurs relationnels;
  • les opérateurs booléens.

1.6.1 Les opérateurs arithmétiques

Les operateurs arithmétiques
+ l'addition
- la soustraction
* la multiplication
/ la division
% le modulo (pour les entiers seulement)

1.6.2 L'arithmétique avec des variables de types différents

En C++, la précision d'un nombre dépend de son type.
Lorsque nous utilisons plusieurs types, nous suivons cette hiérarchie :

Hiérarchie des types numériques
double 8 octets
float 4 octets
int ou long 4 octets
short 2 octets
char 1 octet

Il y a priorité des réels sur les entiers et une priorité des nombres plus longs sur les nombres plus court.

e. Par exemple, pour diviser 49 par 12.5 on convertira tout d'abord 49 en double, soit 49.0, puis on fera la division. Le résultat sera donc de type double.

Si nous affectons une variable float à un int ou un double à un float, il y a perte de donnés possible.

1.6.3 Les opérateurs d'affectation

L'opérateur d'affectation de base est '=', mais nous trouvons également des opérateurs d'affectation arithmétiques ou opérateurs raccourcis
Exemple : Accumul = Accumul + 10 == Accumul += 10;

Les opérateurs d'affectations sont les suivants:
= affectation simple
+= addition
-= soustraction
*= multiplication
/= division
% modulo (pour entier seulement)

1.6.4 Les opérateurs d'incrémentation et de décrémentation

Les opérateurs d'incrémentation permettent de raccourcir certaines opérateurs.
Par exemple, au lieu d'écrire : Compteur = Compteur + 1;
Nous pouvons écrire = Compteur++ ou ++Compteur.

Les opérateurs d'(in/dé)-crémentation :
++ pour l'addition, soit l'incrémentation
-- pour la soustraction, soit la décrémentation
Lorsque l'opérateur précède le nom de la variable il est exécuté avant toutes les autres opérations écrites sur la même ligne (dans le même énoncé), sinon son exécution se fait après toutes les autres opérations. (p.26)

Les opérateurs relationnels et les opérateurs de comparaison

Les opérateurs relationels/comparaisons servent pour les conditions.
La valeur de retour est de type bool, soit 1 si c'est vrai et si c'est faux, 0.
En C, faux == 0 tandis que vrai != 0, il est toutefois possible de creer deux constantes attribuées à vrai et faux valant respectivement 1 et 0.

Les opérateurs booléens

Les opérateurs booléens permettent de vérifier la relation de plusieurs éléments d'une même expression conditionnelle.

Les opérateurs booléens
&& et
|| ou
! négation
